                              morciej Frantisek Rysanek 5 years ago

                              The "merge" handling of Boost means that first, the sensitivity is increased in a smooth way from 0 to Max, and once it is at Max, Boost is increased, in my case in two 20 dB steps. With a cheap electret mike plugged in, the result is too low (white noise) at any setting below Boost, then suddenly too high (crackles) with either Boost setting.
                              If you don't believe me, try and run something like QAS Mixer in parallel with pavucontrol,and you will see what I mean. Look at how the settings change in QAS Mixer, as you control them from pavucontrol.
                              If the Volume were to be decreased by 20dB as the boost increases by the same amount, I could perhaps find the happy medium. Otherwise, I need an option to control it manually, and set the Boost to 20 dB.
                              I would say automating the boost setting in this way is not helpful. Of course, this is only an issue when using a cheap mike with a typical computer mike input.
                              There are at least two hardware solutions, either: 1. add a 40 dB microphone preamp, or simply 2. Go to a thrift store and get a nice USB mike. I did the latter, so this is now a solved issue for me.

                                                  Victor Gaydov Mod Klaus Kobald 6 years ago edited

                                                  Yes, you can set the volume individually for every sink-input (if you don't use flat volumes?). And PulseAudio will create a new sink-input for every entry being played.

                                                  When an entry is currently playing you can find its sink-input using "pactl list sink-inputs" command. Here is example output:

                                                  Sink Input #56
                                                  Driver: play-memblockq.c
                                                  Owner Module: n/a
                                                  Client: n/a
                                                  Sink: 0
                                                  Sample Specification: float32le 2ch 44100Hz
                                                  Channel Map: front-left,front-right
                                                  Format: pcm, format.sample_format = "\"float32le\"" format.rate = "44100" format.channels = "2" format.channel_map = "\"front-left,front-right\""
                                                  Corked: no
                                                  Mute: no
                                                  Volume: front-left: 65536 / 100% / 0.00 dB, front-right: 65536 / 100% / 0.00 dB
                                                  balance 0.00
                                                  Buffer Latency: 0 usec
                                                  Sink Latency: 40514 usec
                                                  Resample method: speex-float-1
                                                  Properties:
                                                  media.name = "test"
                                                  event.id = "test"
                                                  media.role = "event"
                                                  application.name = "pactl"
                                                  native-protocol.peer = "UNIX socket client"
                                                  native-protocol.version = "32"
                                                  application.process.id = "4798"
                                                  application.process.user = "victor"
                                                  application.process.host = "tripbook"
                                                  application.process.binary = "pactl"
                                                  application.language = "en_US.utf8"
                                                  window.x11.display = ":0.0"
                                                  application.process.machine_id = "be715b5004b936d3e8a244bb53c04210"
                                                  module-stream-restore.id = "sink-input-by-media-role:event"

                                                  The index may be different each time. However, the "driver" field will be "play-memblockq.c" and the "media.name" and "event.id" properties will be the name of the sample cache entry. Using these criterias you can find the proper sink-input programmatically.

                                                  Here is how you can change the volume to 50% of the last sample cache entry playing currently:

                                                  pactl set-sink-input-volume "$(pactl list short sink-inputs | grep play-memblockq.c | awk '{print $1}')" 0.5

                                                  (you should run this command while the entry is still being played)
